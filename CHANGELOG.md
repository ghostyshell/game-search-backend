# Changelog

All notable changes to game-search-backend are documented here.
Format: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), adheres to [SemVer](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Full RAWG metadata enrichment. `EnrichGame` now fetches the `/games/{id}` detail endpoint after the search match and persists the full metadata set, not just the 5 list fields it kept before: plaintext description (prefers `description_raw`, falls back to HTML-stripped `description`), metacritic score + url, ESRB rating, developers, publishers, tags, store links (name + url), screenshots (from the list `short_screenshots` plus detail `background_image_additional`, no extra sub-endpoint call), playtime, and reddit_url. New `Game` fields: `rawg_description`, `rawg_metacritic` (pointer so 0 != absent), `rawg_metacritic_url`, `rawg_esrb`, `rawg_developers`, `rawg_publishers`, `rawg_tags`, `rawg_stores` ([]RawgStore{name,url}), `rawg_screenshots`, `rawg_playtime`, `rawg_reddit_url`, `rawg_detail_fetched` (gate). `RawgFields` moved to `models` so the persistence interface references it without importing the metadata service (no persistence->service coupling). `SetGameEnrichment` takes `models.RawgFields` instead of 6 positional args. Cost: 2 RAWG calls per game (search + detail), so backfill consumes key budget against the per-key monthly limit; `ErrAllKeysExhausted` still stops the run cleanly.
- RAWG backfill via the existing `ragenrich` cmd (no new binary). `GetGamesMissingRawgEnrich` now selects `enriched_rawg != true OR rawg_detail_fetched != true`, so a single `ragenrich` run re-enriches every existing entry that pre-dates the detail fetch, and the scheduled `metaEnricher` job keeps new ingests detailed continually. `UpsertGame`/`upsertGamesChunk` seed `rawg_detail_fetched: false` on insert so the missing-query is selective from first write. On a non-exhausted detail error the run skips the game without marking it fetched, so the widened query re-selects it next tick. Operator runs `ragenrich` locally against prod Mongo per the existing runbook; keys exhaust -> stop -> append fresh key -> re-run until no progress.
- dlpsgame.com source-page links. New `LinkTypeSource` (`link_type="source"`) on `GameLink` represents the source site's post/page URL (not a download), so a game detail page can link back to the dlpsgame.com post (mirrors, screenshots, install notes). The DLPsGame scraper now emits one source link per post (`Host="dlpsgame.com"`, `URL=<postURL>`, distinct `_id` from the magnet/DDL keys so no collision) so new ingests capture it going forward. A one-off gitignored `cmd/dlpsgamesourceurl` backfill reconstructs `https://dlpsgame.com/<slug>/` for every existing dlpsgame game (PS3/PS4/PS5), HEAD-verifies it (200), and bulk-upserts a source link per game (idempotent via `SourceLinkSlugs`; 8-way bounded concurrency). New mongo helpers: `GetGamesBySource` (walk a source's games), `SourceLinkSlugs` (slugs already carrying a source link, for the skip set). Frontend renders "View on dlpsgame.com" next to "View on RAWG" and excludes source links from the download list.

### Added
- Repo scaffold: shared cross-repo infra (monorepo context, hooks, AGENTS.md generation), dual remotes (origin akshatsinghkaushik / alt ghostyshell), changelog pre-commit enforcement.
- Backend skeleton: config + ValidateEnvironment (RAWG/TheGamesDB optional), slim Mongo client (games/game_links/debrid_hosts, $set/$setOnInsert upserts, atomic health), SSRF-hardened safe HTTP client, scraper Service + Minerva source (Xbox 360 browse + collection torrents), debrid host sync (TorBox + Real-Debrid public host lists), JobScheduler with TryLock skip-overlap, games/search/browse/debrid/hosts/health handlers, multi-stage Dockerfile (GHCR deploy), parser + tagger self-checks.
- RAWG multi-key enrichment: RAWG_API_KEYS pool (comma-separated, + bare RAWG_API_KEY) rotated around the per-key request limit; RawgClient marks a key exhausted on 429/limit and rotates, returning ErrAllKeysExhausted when the pool is drained so ragenrich stops cleanly and the operator can append a fresh key + re-run. Enricher searches by cleaned title + platform-slug filter, picks best match (exact-on-platform > exact > on-platform > first). MetaEnrich job + cmd/ragenrich one-shot; idempotent via enriched_rawg. 200ms inter-call delay avoids per-second throttles being misread as the monthly limit. Bumped bulk-write op timeout 20s -> 120s so large WAN bulk writes don't silently drop.
- r-roms source (PS2, PSP, PSVita): parses the r-roms megathread pages (regex over server-rendered archive.org/download/<id> hrefs, no HTML-parser dep), enumerates each collection's "original" files via the archive.org Metadata API, and emits one DDL GameLink per ROM file. Runner.RRomsSync + cmd/rromsingest one-shot; shared ingestPlatform path refactored out of MinervaSync. UpsertGameLinks chunked at 2000 so a large platform ingest stays under the 16MB wire limit. RAWG 429 now retries the same key once (1s backoff) before declaring it exhausted, so a transient per-second throttle isn't misread as the monthly limit; dropped the blanket 200ms inter-call delay.
- Bulk game upserts: ingestPlatform now builds the game batch and upserts via one chunked UpsertGames BulkWrite instead of one WAN round-trip per game. A large platform ingest (r-roms PS2 is thousands of games) previously did thousands of sequential round-trips to the shared Sliplane Mongo and was prone to a mid-run connection reset; now it's a handful of chunked BulkWrites, far less exposed to a reset and much faster.
- FitGirl source (PC): two-level scraper - paginates the fitgirl-repacks.site index (/page/N/, WordPress 404 marks end-of-pages), fetches each per-game post (bounded 6-way concurrency), and emits one magnet + one DDL GameLink per distinct filehost per game (part1 representative; multi-part archives need all parts, the frontend resolves each). En/em dashes in scraped titles normalized to ASCII hyphen. Runner.FitGirlSync + cmd/fitgirlingest one-shot; upserts per index page so the long ingest is resumable. Extracted a shared ingestLinks helper from ingestPlatform (single-page and multi-page sources now share the group-by-slug -> bulk-upsert path).
- Minerva PS3: Minerva now also covers PlayStation 3 (Redump browse - 4787 per-title ROM DDLs + a Redump PS3 collection torrent), in addition to Xbox 360. MinervaSync loops both platforms. Complements DLPsGame on PS3 (Minerva gives per-title Redump DDLs, DLPsGame gives filehost mirrors).
- NSWGame source (Switch): two-level scraper for nswgame.com - paginates /category/switch/page/N/ (404 = end-of-pages), fetches each per-game post (bounded 6-way concurrency), and emits one DDL GameLink per distinct filehost per game. Game pages are filtered from nav by slug (every game slug contains "nintendo-switch"); filehost mirrors are extracted as external <a> links minus a nav/social/sibling blocklist (megaup.net, 1fichier.com, send.now, gofile.io, datanodes.to, vik1ngfile.site, ...). Runner.NSWGameSync + cmd/nswgameingest one-shot; upserts per category page so the ~770-page ingest is resumable. Renamed the unused DLPsGameSync config slot to NSWGameSync.
- DLPsGame source (PS2, PS3, PS4, PS5): three-level scraper for dlpsgame.com. The category index (/category/<cat>/page/N/, 404 = end-of-pages) lists per-game posts filtered by a kebab-token platform match (-ps4-) plus a prefix gate that drops admin/index slugs (list-all-game-ps4, list-game-ps5, daily-update, ...) that also carry the token; PS4 slugs carry a format suffix (-ps4-pkg, -ps4-fpkg) so a HasSuffix check would miss them. Each post hides its real links in a Shortcodes Ultimate `secure-data` div whose `data-payload` attribute is base64-encoded HTML; the decoded HTML points at a sister site (downloadgameps3.net/archives/<id>) whose static page holds the plain filehost <a href> mirrors (1fichier, akirabox, buzzheavier, rootz, mediafire, filekeeper, datavaults, vikingfile, gofile, datanodes, ...). Some payloads also carry direct filehost hrefs (no sister hop). The site wraps filehost URLs in an AdLinkFly shortener (clk.sh) loaded by a Dean Edwards packed script, but that rewrite is browser-only; the sister-site hrefs are the raw filehost URLs in the static HTML, so a server scraper needs no JS engine or adblock (the "JS is just ads" intuition holds - adblock is unnecessary server-side). Emits one DDL GameLink per distinct filehost per game (deduped by host across direct + sister), plus any magnet. Runner.DLPsGameSync (calls the DLPsGame scraper directly, not BrowseAll, because PS2/PS3 overlap RRoms/Minerva) + cmd/dlpsgameingest one-shot; upserts per category page per platform so the four-platform WAN ingest is resumable. New DLPsGameSync config slot (DLPSGAME_SYNC_INTERVAL_MS / _INITIAL_MS). Fills the PS4/PS5 gap left by the earlier DLPsGame removal.

### Changed
- Switch + PC browse order now matches the source sites. nswgame (Switch) and fitgirl (PC) scrapers now capture each post's `article:published_time` Yoast meta tag and set `SourcePublishedAt` on every emitted link, so `ingestLinks` propagates the real post publish date to the `Game` as the browse sort key. Previously these sources emitted links with `SourcePublishedAt: 0`, so `ingestLinks` fell back to scrape wallclock (`now`), and within one ingest batch every game shared the same `now` - Switch/PC browse order was BulkWrite/insertion order, not the source's newest-first order. Existing nswgame/fitgirl games also had `source_published_at = updated_at` (scrape wallclock) from the one-time `Migrate` backfill, so they sorted wrong too. A one-off gitignored `cmd/pubdatedfill` backfill reconstructs each game's post URL from its stored slug (`https://nswgame.com/<slug>/` or `https://fitgirl-repacks.site/<slug>/`), GETs it, parses the meta tag, and bulk-updates `source_published_at` in place (idempotent, 8-way bounded). New mongo helper `SetGamesSourcePublishedAt` (chunked bulk `$set` of `source_published_at` only, not `updated_at`). The dlpsgame-specific `dlpsGamePublishedAt`/`dlpsGamePublishedRe` were extracted into a shared `articlePublishedAt` helper (helpers.go) reused by all three WordPress scrapers; `TestArticlePublishedAt` covers it.
- Removed the `dlpsGameSync`, `fitGirlSync`, and `nswGameSync` job registrations from `main.go`. All three sources (dlpsgame.com, fitgirl-repacks.site, nswgame.com) are Cloudflare-fronted and block Sliplane's Hetzner egress (403 / TLS handshake failure), so a prod-side sync errored every tick and never upserted - the catalogs were populated entirely by one-shot Mac bulk-fills and went stale between them. They are now synced from the Mac (which can reach them) via caffeinate-wrapped launchd cron jobs running `cmd/{dlpsgame,fitgirl,nswgame}ingest` against prod Mongo; the cmd tools call `runner.<Source>Sync` directly so they share one code path with the old scheduler entry. `minervaSync` and `rromsSync` are NOT CF-blocked and keep running in prod. The unused `DLPsGameSync`/`FitGirlSync`/`NSWGameSync` config slots are retained (env vars harmlessly ignored).
- RAWG best-match now rejects wrong-game matches: `pickBest` kept a `default: return first` / on-platform fallback that returned the first RAWG search result even when its title shared nothing with the wanted title, so a title RAWG lacks (e.g. "EA Sports UFC 5") was matched to a loosely-related result ("EA Sports FC 24") and the wrong game's cover/description got stored against the right title/links (the user-reported "mismatched names and details for their contents" on the PS5 browse page). The fallbacks are replaced with a Jaccard similarity floor (0.5) over stopword-filtered title tokens plus a shared-token guard: `pickBest` still prefers an exact (case-insensitive) title match on the target platform, then any exact title match, then the highest-similarity result on-platform, then overall, but only returns that fuzzy result when its Jaccard clears the floor AND it shares at least 2 distinctive tokens with the want title when the want has 2+ distinctive tokens (the guard rejects generic-name collisions where a 1-token RAWG name like a chicken game named "Behemoth" hits Jaccard 0.5 on the single shared noun against "Skydances BEHEMOTH"); otherwise returns nil so `EnrichGame` marks the title terminal-empty (honest no-data) instead of wrong-match (wrong cover/description). Tokens keep 2+ char tokens (numbers and codes like "fc"/"24" are the discriminative ones) minus a small stopword set ("the", "of", "edition", "remastered", ...) so generic words don't inflate similarity. New `TestPickBestJaccardFloor` covers the reported wrong-match cases (ufc5/fc24, story-of-seasons/live-a-live, like-a-dragon-gaiden/infinite-wealth reject, skydances-behemoth vs single-token behemoth reject) and legit fuzzy matches (syberia-remastered/syberia, bud-spencer-punctuated accept). Followed by a PS5-targeted re-enrich (ResetEnrichmentForPlatform + ragenrich): all 755 PS5 games re-processed, 26 wrong matches cleared to terminal-empty (with_rawg_id 755 -> 729), the rest kept or corrected their match.
- PS5 re-enrich one-off helpers: `Client.ResetEnrichmentForPlatform` clears the enriched_rawg / rawg_detail_fetched flags for a platform so the widened missing-query re-selects those games and `ragenrich` re-runs the (fixed) search+detail against each; `Client.CountPlatform` snapshots total/enriched/detail-fetched/with-rawg_id for before/after verification. Backfill hook for re-correcting wrong matches from the old pickBest fallback; idempotent. Used by the gitignored `cmd/ps5reset` local tool.
- RAWG no-match titles are now terminal: `EnrichGame` returned `RawgFields{}` (DetailFetched=false) when RAWG had no entry for a title, so `SetGameEnrichment` wrote `enriched_rawg=true` with `rawg_detail_fetched=false`, and the widened missing-query (`enriched_rawg != true OR rawg_detail_fetched != true`) re-selected the same un-matchable titles every run. Obscure Japan-only PS2 exclusives and eShop indies RAWG doesn't catalog looped forever, re-searching each pass and burning key budget indefinitely (this is what stalled the 2026-07 backfill at 59 stragglers with a flat detail count while the run counter climbed). `EnrichGame` now returns `RawgFields{DetailFetched: true}` on a confirmed no-match so the caller writes `rawg_detail_fetched=true` and the missing-query stops re-selecting; the display fields stay empty for an alternate-source one-off to fill later. The detail-error path still returns an error (DetailFetched stays false, retried) - only the confirmed-no-match path is terminal. New `TestEnrichGameNoMatchTerminal` stubs RAWG `/games` with an empty results array and asserts DetailFetched=true. A one-off `cmd/tgdbenrich` (gitignored, local bulk-fill tooling) then backfilled the 59 from TheGamesDB - 13 got cover/description/genres/developers/release date, 46 were marked terminal-empty, draining the missing set to 0 (catalog is now 58319/58319 enriched + detail-fetched) without burning a RAWG key pass.
- Browse sort matches source post order: `BrowseGames` + `SearchGames` now sort by `source_published_at: -1` instead of `updated_at: -1`. `updated_at` is scrape wallclock, so within one ingest batch every game shared the same `now` and the "newest first" order was BulkWrite/insertion order, not the source's order - the frontend PS5 list did not match dlpsgame.com/category/ps5/. The DLPsGame scraper now captures each post's `article:published_time` meta tag (the date dlpsgame.com's category page sorts by) in `scrapePost` and carries it on every `GameLink` as `SourcePublishedAt`; `ingestLinks` propagates it to the `Game` and falls back to scrape wallclock for sources without a publish date (so non-dlpsgame games keep the prior "newest scrape first" behavior under the unified sort). New `Game`/`GameLink` field `source_published_at`. `Migrate` runs a one-time idempotent backfill (`source_published_at = updated_at` where the field is missing) so the existing ~58k games get a sort key without a full re-scrape; dlpsgame games are corrected to the true post date on their next sync. The browse index swaps from `{platform:1, updated_at:-1}` to `{platform:1, source_published_at:-1}`. No frontend change (the field ships via the struct tag); no extra HTTP call (the date is parsed from the post body already fetched by `scrapePost`).

- Scraper review fixes (code-reviewer + security-auditor pass): FitGirl + NSWGame DDL dedup now lowercases the captured host before dedup + storage so an uppercase host isn't double-counted; NSWGame link host regex hardened to case-insensitive. NSWGame + FitGirl per-index-page post fan-out is capped (200) with a log line so a crafted index page can't spawn unbounded goroutines. BrowseAll forwards partial links alongside a per-scraper error (Minerva collection torrents were silently dropped on a transient browse-page failure). FitGirlSync / NSWGameSync / ingestPlatform distinguish a genuine end-of-pages (no links, no errors) from a transient scrape failure (no links + errors) and surface the latter as an error so a failed page 1 isn't a silent empty ingest. ingestLinks has a central http/https/magnet scheme gate (defense-in-depth on the persist path). SearchGames neutralizes user regex metacharacters (QuoteMeta + ^ anchor) so a search query can't be crafted into a ReDoS / pattern injection. CORS prod default is fail-closed (deny cross-origin when no origin is allowlisted) instead of a wildcard, and a wildcard config echoes the literal `*` without Allow-Credentials so it can't become an open origin reflector. FitGirl index page now validates article href host against the site base (matches the NSWGame host guard; the safe dial layer already blocks non-public IPs, this adds application-layer defense-in-depth so a compromised index page can't steer per-game fetches).
- DLPsGame covers PS3/PS4/PS5 only (PS2 dropped): dlpsgame.com PS2 posts use a plain-body link structure (no base64 data-payload; sister-site archive pages are dead/empty for PS2) that the 3-hop scraper doesn't parse, so every DLPsGameSync run no-op'd PS2 ("no links, stopping" on page 1). The category is niche gravure/idol content, and PS2 is owned by RRomsSync per the source plan (r-roms already populates PS2 in prod - 50+ games, 0 from dlpsgame). Removed PS2 from DLPsGameScraper.Platforms(), dlpsGameCatSupported, and the DLPsGameSync platform loop; building a body-parser for a niche category already covered elsewhere was YAGNI. PS2 still ingested via RRomsSync.
- RAWG monthly-limit detection: RAWG signals the per-key monthly limit as HTTP 401 with body `{"error":"The monthly API limit reached"}`, not 429. The RawgClient rate-limit predicate only matched 429/402 or a body of "Request limit reached", so a monthly-limited key was never marked exhausted - it fell through to the plain HTTP-error path, the enricher logged the 401 per game and moved on, and pickKey kept handing back the same dead key until the 100-game stall-break fired (wasting ~100 calls per run). The predicate now also matches a body containing "monthly API limit reached", so a monthly-limited key is marked exhausted and the pool rotates (or returns ErrAllKeysExhausted cleanly when drained). Gated on the body string, not bare 401, so an actually-invalid key (401 with a different body) still surfaces as a plain error.
- DLPsGame targeted re-run + sister cap: DLPSGAME_PLATFORMS env (comma-separated, e.g. `ps5,ps2`) narrows DLPsGameSync to a subset of the four platforms, in stable PS2/PS3/PS4/PS5 order. Empty/unset = all four (the scheduled job's behavior unchanged). Lets the one-shot cmd/dlpsgameingest fill PS5 / retry PS2 without re-scraping PS4's ~260 pages first (which would re-hit the 4h ctx deadline before reaching the missed platform). Unknown names are dropped with a log line; an all-unknown set falls back to all four so a typo can't silently run nothing. DLPsGame sister-site fetch cap raised 16 -> 32: real outlier posts legitimately carry 17-26 sister archive pages (big multi-part games - WWE 2K24 26, RDR2 25, WWE 2K25 21, Disney Dreamlight Valley 18), so 16 dropped real filehost mirrors; 32 recovers them while staying bounded (32 x 30s worst case per post). A re-run backfills the previously-capped games (idempotent per-page upsert).
- DLPsGame rate-limit handling: dlpsgame.com 429s under burst scraping, and the 3-hop scraper is fetch-heavy (1 post + up to 16 sister pages per game), so a bulk fill trips the limiter and aborts PS4/PS5 for the run. fetchURLStatus now retries 429 with an increasing backoff (3s, 6s, 12s, ctx-cancellable) so a background fill progresses politely instead of aborting a platform on the first 429; 5xx are still surfaced immediately (DLPsGameSync logs + continues to the next platform). Per-game fan-out reduced 6 -> 4 to lower burst pressure (the 429 backoff is the safety net).

### Removed
- Debrid "drop unsupported links" filter: intentionally NOT implemented. The plan called for dropping links whose host is in neither provider's host list, but minerva-archive.org (per-title ROM DDL, the bulk of Xbox 360 + PS3 links) is not a TorBox/RD "hoster" yet is directly downloadable (no wait timer) - dropping it would nuke per-game DDL and leave only giant collection torrents. SupportedBy encodes debrid eligibility (empty = not debrid-eligible); the frontend offers a direct-download fallback for such DDL links. Verified host coverage: megaup.net, 1fichier.com, datanodes.to, send.now, gofile.io, vik1ngfile.site, fuckingfast.co, archive.org, mega.nz, mediafire.com are all debrid-supported (mostly TorBox); only multiup.io and minerva-archive.org are not.

### Deferred

TheGamesDB would add retro cover art redundancy; not worth the second enrich pipeline + key in v1. Add if RAWG cover coverage for retro consoles proves thin.
